代码管理中常用的git命令

首先新建远程仓库
git clone git@github.com:username/projectname.git（下载远程代码到本地）
git init (初始化本地仓库)
git add .(将本地文件加入缓存区)
git commit -m “修改的内容说明” （说明性内容）
git push origin master （push文件到仓库中）

之后每次在提交之前都需要先pull一下（先更新，再提交）
git pull (拉取远程代码，使本地跟远程保持一致)

git remote add origin git@github.com:username/projectname.git（本地仓库链接远程仓库）

git checkout branch-name (切换到某分支上进行开发)
一个模块完成后，把代码提交到线上。执行
git add .
git commit -m ‘ajax’
git push //提交代码到分支
git checkout master //切换到主分支
git merge branch-name //执行合并
git push //将主分支代码提交

添加/修改多个文件夹/文件：
git add .
git commit -m “add screenShorts”（说明性内容）
git push -f origin master

屏蔽文件：可直接创建.gitignore文件，在里面写入需要屏蔽的文件(最常见的Mac系统自带的.DS_Store文件)
如果.DS_Store文件已被不小心上传至github，我们用以下方法可以很好的解决这个问题：
git rm -r –cached some-directory
git commit -m ‘Remove the now ignored directory “some-directory”‘
git push origin master

git的安装：
通过brew安装git,命令如下:brew install git
检测git是否安装成功：git –version
显示当前git的路径：which git（一般在/usr/local/Cellar/git）
